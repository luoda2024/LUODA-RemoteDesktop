# 🚀 LUODA CI 构建快速启动指南

## ⚠️ 重要说明

由于 Gitee API 限制，**流水线触发必须通过 Web 界面手动操作**。以下是完整的操作步骤。

---

## 📋 步骤 1：手动触发流水线（必须）

### 访问流水线页面

打开浏览器访问：
```
https://gitee.com/soulemo_1/dicad/pipelines
```

### 触发构建步骤

1. **点击「运行流水线」按钮**（页面右上角）

2. **选择流水线**
   - 在下拉列表中选择 `luoda-full-build`
   - 这是完整的构建流程（EXE + MSI + APK + DEB）

3. **选择分支**
   - 分支：`master`
   - 标签：（留空）

4. **点击「运行」按钮**
   - 系统会开始排队构建
   - 通常 1-2 分钟内开始

---

## 📊 步骤 2：监控构建进度

### 方法 A：Web 界面监控（推荐）

1. **在流水线列表页面**
   - 查看最新一行（ID 最大的）
   - 状态会实时更新

2. **点击流水线 ID**
   - 进入详情页
   - 可以看到每个阶段的进度

3. **查看日志**
   - 点击每个阶段（如 `build_windows`）
   - 右侧显示实时日志
   - 日志每几秒自动刷新

### 方法 B：使用监控脚本

```bash
# 运行监控脚本
cd /workspace/LUODA-RemoteDesktop
python3 auto-build-monitor.py
```

脚本会：
- 显示最新流水线状态
- 每 10 秒自动刷新
- 构建完成时通知

### 方法 C：使用 Web 监控面板

```bash
# 在浏览器打开
# Windows: start monitor.html
# Mac: open monitor.html
# Linux: xdg-open monitor.html
```

## 🔧 常见错误处理

### 错误 1：找不到流水线

**现象**：访问页面显示空白或"未找到"

**解决**：
1. 确认仓库 URL 正确：`https://gitee.com/soulemo_1/dicad`
2. 检查是否已登录正确的 Gitee 账号
3. 确认账号有该仓库的访问权限

### 错误 2：流水线无法运行

**现象**：点击"运行"按钮无反应

**解决**：
1. 检查 Gitee 流水线配额是否用完
2. 刷新页面重试
3. 检查 `.workflow/luoda-full-build.yml` 语法

### 错误 3：构建失败

**现象**：状态变为 ❌ failure

**解决**：
1. 点击流水线查看具体失败阶段
2. 查看该阶段的日志找出错误原因
3. 常见错误：
   - 依赖缺失 → 检查版本要求
   - 内存不足 → 等待其他构建完成
   - 网络超时 → 重试构建
